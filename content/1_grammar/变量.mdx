# 变量与输入语句cin

## 数据和数值

　　在计算机中，数据是可以由计算机移动、处理或存储的任何信息。

　　程序可以通过多种方式获取要使用的数据，可以从磁盘里读，可以从内存里找，甚至可以去网上爬取。

　　我们把单个数据称为值。值的常见示例包括字母（例如`a`）、数字（例如`5`,`23`）和文本（例如`Hello`）。

## 随机存取存储器RAM

　　计算机中的主存储器称为随机存取存储器（通常简称为**RAM**）。当我们运行一个程序时，操作系统将程序加载到**RAM**中。任何编码到程序本身的数据（例如“`Hello, world!`”等文本）都会在此时加载。

　　我们不是数字生命，所以无法直接进入到存储器中寻找到想要的值，但我们可以通过变量的形式，通过变量来存取和改变一些数值

## 变量

### 什么是变量
　　变量是一个存储值的存储区域，我们可以通过变量来存取一些需要的值。

　　可以想象这样的场景，你喊来了许多机器人，当你需要记下来一个数字时，你喊来一个机器人，告诉他：“**记下来数字`23`！**”，一会，你又问机器人：“**刚刚让你记的数字是多少？**”

　　你并不用管机器人是如何记下来的，只需要管机器人存取数字就行了，对于变量也一样，你并不用管变量把数字存在了哪，只需要对他索要和存储即可。

### 声明一个变量

　　为了创建变量（喊来机器人），你需要学习**声明语句**，我们也叫做**定义**，下面是定义一个名叫x的变量的例子：

```cpp
int x;
```

<Info title="请注意">
	定义的变量名往往不能是单词，因为很多单词已经变成了编程工具的名字，所以最好避开一些简单的单词来作为变量名。

	在比赛中，我们往往采用
	- 简单的字母(例如`a`,`b`,`n`,`x`)
	- 单词的缩写(例如`cnt`代表count，或是`tot`代表total)
	来当作变量名，用来避免这种情况。
</Info>

　　当程序运行时，x就是一个可以用于存储一条数据的变量了，我们可以随时读取变量中的数据，也可以使用赋值运算符`=`对变量存储的数据进行修改，例如：

```cpp
int x;	//声明一个变量x
x=1;	//给x存储1
int y=2;	//声明一个值为2的变量y
int a,c=2;	//声明一个变量a，值为2的变量c
x=3;	//让x存储3
a=x;	//让a的值和x存储的值一样
```

　　在上面这段代码中，我们一共创建了4个变量，分别是`x`，`y`，`a`，`c`,他们分别存储`3`,`2`,`3`,`2`四个值。

　　其中`x`一开始没有值，我们向它存储了数字`1`，赋值的数字`3`会把之前的数字`1`覆盖，最后我们还把`x`中的值也同样复制到了`a`里。

<Info title="请注意！">
	定义的变量如果没有赋值过，那它的值往往是随机的。
</Info>

### 数据类型
　　然而，变量并不是什么数据都能存储，我们先介绍几种不同的数据类型。

- int 只能存储没有小数点的整数，但是能存储的值大约是$-2*10^8到2*10^8$的所有数字，也就是刚刚我们使用的变量
- long long 也用于存储整数，但是能存储的值比int大得多,大约是$-4*10^{18}到4*10^{18}$之间
- ~~float~~ 用于存储小数（也叫浮点数），但是由于精度太低，在比赛中已经不再使用
- double 用于存储小数（也叫浮点数）
- char 用于存储**单个的**字符，例如`c`,`A`,`!`等
- string 用于存储**多个的**字符组成的字符串，例如"`hello world！`"

```cpp
int n=23;
double pi=3.14
char c='!';
strng s="hello world!";
```

<Info title="char和string应该用哪个">char和string有比较大的区别(在这里表现为一个用单引号，一个用双引号)，这里我们后面再聊</Info>

### 定义多个变量
　　如果想定义多个变量，可以多次写声明语句，也可以用逗号分隔开，下面两个代码实际上是相同的。
```cpp
int a,b;
```
```cpp
int a;
int b;
```

当以这种方式定义多个变量时，新同学往往会犯两个常见错误，例如
```cpp
int a, int b; // 错误的写法，写了两个int

int a, b; // 正确的写法
```
```cpp
int a, double b; // 错误的写法，把两条语句写在一起

int a; double b; // 正确的写法，用分号分隔开了两条语句

// 下面是更加正确的写法，更有可读性
int a;
double b;
```
